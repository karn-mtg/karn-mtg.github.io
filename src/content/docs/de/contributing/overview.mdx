## Bevor du anfängst

Für alles, was über kleine Korrekturen hinausgeht, öffne bitte zuerst ein Issue oder eine Diskussion, um den Ansatz abzustimmen. Das spart Zeit für dich und die Maintainer.

Bei Bugfixes und Dokumentationsverbesserungen kannst du direkt mit einem Pull Request einsteigen — kein Issue nötig.

## Eine gute Fehlermeldung verfassen

Füge Folgendes ein:

1. **Schritte zur Reproduktion** — die genaue Abfolge von Aktionen, die den Fehler auslöst.
2. **Erwartetes Verhalten** — was du erwartet hast zu sehen.
3. **Tatsächliches Verhalten** — was tatsächlich passiert ist. Screenshots helfen.
4. **Umgebung** — dein Betriebssystem, die OS-Version und die Karn Forge-Version (sichtbar unter Einstellungen → Über).
5. **Logs** — wenn die App abgestürzt ist, kopiere den Fehler aus **Einstellungen → Logs** oder der Systemkonsole.

## Pull Request-Richtlinien

- Halte Änderungen fokussiert — ein PR pro logischer Änderung.
- Schreibe eine klare PR-Beschreibung, die erklärt, *was* sich geändert hat und *warum*.
- Folge dem vorhandenen Code-Stil. Das Projekt verwendet ESLint und TypeScript strict mode — stelle sicher, dass `npm run lint` und `npm run type-check` bestehen, bevor du einreichst.
- Füge Tests hinzu, wenn du Logik in `domain/` oder `application/` änderst. UI-Änderungen erfordern keine Tests, sollten aber manuell überprüft werden.
- Das Canvas (`DeckView.tsx`) ist der komplexeste Teil der Codebasis. Wenn du es anfasst, lies zuerst die [Architektur](/contributing/architecture)-Anleitung.

## Verhaltenskodex

Dieses Projekt folgt einem einfachen Standard: Sei respektvoll, sei hilfsbereit und gehe von guten Absichten aus. Beiträge von allen sind willkommen, unabhängig vom Erfahrungsstand.

## Was im Umfang liegt

| Im Umfang | Außerhalb des Umfangs |
|---|---|
| Bugfixes | Änderungen, die ein immer laufendes Backend erfordern |
| Performance-Verbesserungen | Monetarisierungs-Funktionen |
| Neue Widget-Typen | Plattformspezifische Funktionen, die nur auf einem OS funktionieren |
| Kartendetail-Erweiterungen | Kartendaten scrapen oder weiterverteilen |
| UI-Verbesserungen | |
| Verbesserungen der Barrierefreiheit | |
| Neue unterstützte Import-Formate | |
