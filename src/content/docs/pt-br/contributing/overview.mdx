## Antes de começar

Para qualquer coisa além de pequenas correções, abra uma issue ou discussão primeiro para alinhar a abordagem. Isso economiza tempo tanto para você quanto para os mantenedores.

Para correções de bugs e melhorias na documentação, você pode ir direto para um pull request — sem necessidade de abrir uma issue antes.

## Fazendo um bom relatório de bug

Inclua o seguinte:

1. **Passos para reproduzir** — a sequência exata de ações que desencadeia o bug.
2. **Comportamento esperado** — o que você esperava que acontecesse.
3. **Comportamento real** — o que realmente aconteceu. Screenshots ajudam.
4. **Ambiente** — seu SO, versão do SO e versão do Karn Forge (visível em Configurações → Sobre).
5. **Logs** — se o app travou, copie o erro de **Configurações → Logs** ou do console do sistema.

## Diretrizes para pull requests

- Mantenha as alterações focadas — um PR por alteração lógica.
- Escreva uma descrição clara do PR explicando *o que* mudou e *por quê*.
- Siga o estilo de código existente. O projeto usa ESLint e TypeScript em modo strict — garanta que `npm run lint` e `npm run type-check` passem antes de enviar.
- Adicione testes se estiver alterando a lógica em `domain/` ou `application/`. Alterações de UI não precisam de testes, mas devem ser verificadas manualmente.
- O canvas (`DeckView.tsx`) é a parte mais complexa do codebase. Se for mexer nele, leia o guia de [Arquitetura](/pt-br/contributing/architecture) primeiro.

## Código de conduta

Este projeto segue um padrão simples: seja respeitoso, seja prestativo e assuma boa-fé. Contribuições de todos são bem-vindas independentemente do nível de experiência.

## O que está no escopo

| No escopo | Fora do escopo |
|---|---|
| Correções de bugs | Alterações que requerem um backend sempre ativo |
| Melhorias de desempenho | Recursos de monetização |
| Novos tipos de widgets | Recursos específicos de plataforma que só funcionam em um SO |
| Melhorias nos detalhes de cartas | Raspagem ou redistribuição de dados de cartas |
| Polimento de UI | |
| Melhorias de acessibilidade | |
